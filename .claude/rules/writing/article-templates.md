# 記事構成テンプレート集

## 概要

技術記事作成時の参考テンプレート。記事のタイプに応じて選択する。

技術記事の新規作成時は以下のパターンから選択する。コラム・雑記の場合は内容に応じて柔軟に調整すること。

## パターンA：問題解決型記事（最頻出）

トラブルシューティングや課題解決を目的とした記事。

```markdown
## 背景

（問題の提起、動機の説明、なぜこの記事を書くのか）

## 課題 / 問題 / 状況

（具体的な課題の特定、現状の問題点）

## 解決方法 / 実装

（具体的な解決策の提示）

### 基本的な実装

（最もシンプルな実装例、コード5個以上推奨）

### 応用例

（実践的な応用パターン）

### エッジケースへの対応

（特殊なケースや注意点）

## 検証 / デモ / 動作検証

（実際の動作確認、サンプルリポジトリへのリンク）

## 参考 / まとめ

（参考文献、結論、要点の再確認）
```

## パターンB：知識共有型記事（2023年以降増加）

技術仕様や概念の説明を目的とした記事。

```markdown
## 概要

（技術や概念の要約、この記事で扱う内容）

## [技術名]とは / [技術名]について

（技術の定義、基本的な説明）

## 記述方法 / 使い方

（具体的な使用方法、構文説明）

## デモ / 実装例

（実際の使用例、コード例）

## 参考 / まとめ

（参考文献、要点の再確認）
```

## パターンC：ハウツー型記事

特定の手順や方法を示す記事。

```markdown
## やりたいこと / 背景

（目的の明確化、何を実現したいか）

## 環境 / 前提

（必要な環境、前提条件）

## 方法 / 実装

（具体的な手順、コード例）

## 参考

（参考文献）
```

## 構成の特徴

- 論理的な流れ（問題提起 → 背景 → 解決策 → 実装 → 検証 → 結論）
- セクションの分離（1つの見出しに複数の内容を詰め込まない）
- 段階的な深化（基本 → 応用 → エッジケースの順で説明）

## 活用方法

1. 記事作成時に最も適したパターンを選択
2. テンプレートをベースに、必要なセクションを調整
3. コラム・雑記の場合は柔軟に適用
