# 文章の声（Writing Voice）

技術メモ寄りで、簡潔、断定的、主観が薄い。レビューコメントや検証メモをそのまま読みやすい記事にした文体である。

## 文体原則

1. **断定する** —「〜である」「〜になる」「〜を使う」で言い切る
2. **現象として書く** — 主観（「気になる」「確認したい」）より事実・結果をそのまま置く
3. **検証結果として締める** — 判断より観測結果ベースで締める

## 推奨する言い回し

| 言い回し         | 役割                                 | 使用場面                   |
| ---------------- | ------------------------------------ | -------------------------- |
| 〜である         | 説明を短く言い切る                   | 基本の断定                 |
| 〜が必要である   | 主観ではなく条件として書く           | 要件・前提の記述           |
| 以下の通りである | 説明らから具体例へ切り替える           | 箇条書き・表・設定例の直前 |
| 〜が発生する     | 問題提起を感想ではなく現象として書く | 問題・副作用の記述         |
| 〜を実装する     | 解決策を実務的に示す                 | 方法・手順・対処の説明     |
| 〜を組み合わせて | 構成要素の関係を簡潔に示す           | 複数技術の併用             |
| 〜を活用する     | 手法の採用をやや整理された語感で表す | 既存の仕組みを使う文脈     |
| 〜を推奨する     | 強すぎず判断を示す                   | 結論・注意点でのお勧め     |
| 確認できなかった | 観測結果として否定する               | 実測・検証ベースの否定     |
| 差分は生じない   | 比較結果を短く明確に言う             | 比較記事・移行記事         |

## 避けるべき表現

| 避ける                                   | 理由         | 代替                                                                                                                                          |
| ---------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| 〜だろう / 〜かもしれない / 〜と思われる | 主観的で曖昧 | 断定するか、条件を明示する                                                                                                                    |
| 気になる / 確認したい                    | 感想寄り     | 「確認点は以下の2点である」                                                                                                                   |
| 焦点になる / 帰結がある                  | 抽象的       | 文脈別に: 因果 =「〜につながる」「〜を引き起こす」 / 観測 =「差分の有無を確認する」「次の2点が成り立つ」 / 結論 =「〜である」「〜に寄与する」 |
| 積極的な理由は見当たらない               | 評価寄り     | 「優位性は確認できなかった」                                                                                                                  |

## 文章構成の原則

- **導入**: 前置きで語りすぎない。早めに「何を検証したか」「何が課題か」に入る。「本章では〜」「〜について説明する」「〜を紹介する」等の前置きは削除する (具体例は [editing-examples.md](./editing-examples.md) の「AI表現の削除」節）
- **本文**: 問題提起 → 確認点 → 結果の流れを素直に組む
- **まとめ**: 評価より結果寄りで締める。観測結果をそのまま書く
- **語彙**: 抽象語より実務語を使う。何を確認する節なのかがすぐ分かる表現を選ぶ
